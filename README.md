# Qureka : 자기주도 학습을 하는 대학생을 위한, 강의자료 기반 개인 맞춤형 소크라테스식 질문을 생성하여 사고의 확장을 유도하는 AI 튜터링 서비스
---
## ✨ 서비스 개요
PDF 강의록, 학생의 손필기, 강의 녹음 파일에 대한 종합적인 분석을 기반으로 비판적 사고 질문과 소크라테스식 대화를 생성하여, 학생의 깊이 있는 학습을 돕는 능동적 학습 파트너로서 기능하는 AI 조교 웹 서비스이다.

---
## 🚀 주요 기능

### 🔹 1. 강의 자료 업로드 및 분석
- 강의 **PDF**, **손필기 이미지**, **음성 녹음 파일**을 업로드하면  
  AI가 자동으로 텍스트를 추출하고 분석하여 학습 기반 데이터를 생성


### 🔹 2. AI 기반 소크라테스식 질문 생성
- 업로드된 자료의 의미를 분석하여 **벡터 데이터**로 구조화하고,사용자 수준에 최적화된 **소크라테스식 발문**을 생성하여 심층 학습 유도  
- 학습자에게 정답을 즉시 제공하는 대신, 사용자 답변을 분석한 후 후속 질문을 제시함으로써
  스스로 논리적 모순을 깨닫고 개념을 확장하게 돕는 유도형 학습 세션을 제공
- 단순한 질의응답이 아닌 **지속적이고 맥락 있는 학습 대화**를 구현


### 🔹 3. 대화형 학습 인터페이스
- `React.js` 기반의 웹 UI에서 **채팅형 학습 환경**을 제공
- 사용자는 AI와 자연스럽게 대화하며, **자기주도적으로 개념을 확장하고 이해를 심화**  


### 🔹 4. 개인 맞춤형 학습 리포트
- 학습 대화 로그를 분석해  
  **사고 경로 및 논리 구조를 시각화한 요약본**을 제공  
- 학습자는 자신의 사고 흐름과 이해 과정을 점검하고  
  **개인화된 학습 피드백**을 받을 수 있음  


### 🔹 5. 통합형 학습 환경
- 다양한 학습 자료를 한곳에서 관리할 수 있는 **원스톱 학습 플랫폼**을 제공  
- `PostgreSQL`과 `pgvector`를 통해  
  사용자의 학습 이력과 벡터 임베딩을 관리하여  
  **맞춤형 학습 경험**을 지속적으로 개선

## 🔧 적용 기술

### 1. FrontEnd
- React.js : 사용자 인터페이스(UI) 개발, 대화 화면, 자료 업로드/조회 기능 구현
### 2. BackEnd
- FastAPI : 비동기 처리를 지원하는 API 서버 구축, AI 모델 연동 및 데이터 처리 로직 관리
- SQLAlchemy : 파이썬 객체 기반의 DB 조작(ORM) 및 데이터 무결성 관리
- PyJWT / Argon2 : 사용자 보안 인증 및 비밀번호 암호화 관리
### 3. DataBase 
- PostgreSQL : 사용자 계정 정보, 강의 메타데이터, 대화 로그 관리
- pgvector : 고차원 임베딩 벡터 저장 및 유사도 검색(L2 Distance)을 통한 컨텍스트 추출 최적화
### 4. AI & Data Engineering
- Google Gemini 2.5 Flash : 비전 기반 스마트 OCR(손글씨 인식) 및 소크라테스식 학습 대화/피드백 생성
-	Gemini-Embedding-001 : 강의 텍스트 데이터를 벡터화하여 지능형 검색이 가능한 형태로 변환
-	OpenAI Whisper : 강의 음성 파일(STT) 분석 및 텍스트 데이터 추출
-	PyMuPDF (fitz) : PDF 문서의 시각적 분석을 위한 페이지별 이미지 렌더링 및 전처리 수행

## 📌 핵심 기술 요소 

### 1. RAG
- 고정밀 벡터 인덱싱 (Indexing) : Gemini-Embedding-001을 활용해 학습 자료를 3072차원의 고밀도 벡터로 변환하고, 페이지 번호 메타데이터와 함께 pgvector에 저장하여 정밀한 검색 기반 마련
- 의미론적 컨텍스트 검색 (Retrieval) : 사용자의 질문을 실시간 벡터화하여 DB 내 최적의 텍스트 조각을 L2 Distance(유클리드 거리) 알고리즘으로 추출하며, 직접 구현한 쿼리 로직을 통해 검색 속도 최적화
- 근거 기반 질문 생성 (Grounded Generation) : 검색된 강의 컨텍스트 내에서만 답변하도록 제한(Constraint)을 걸어 AI의 환각(Hallucination)을 방지하고, 강의 내용에 철저히 기반한 소크라테스식 발문 구현

### 2. 프롬프트 엔지니어링
- 단계적 사고 유도 프롬프트 구조화 : 정답을 직접 알려주지 않고 [힌트 제공] → [논리 확인] → [반증 질문]으로 이어지는 시스템 프롬프트를 설계하여 학습자의 능동적 사고 유도
- 사용자 맞춤형 페르소나 주입 : 학생의 학과 및 학년 데이터를 프롬프트에 동적으로 결합하여, 학습자의 수준에 최적화된 전문 용어와 문체를 사용하는 개인화된 튜터링 구현
- 세션 기반 맥락 유지 (Memory) : 이전 대화 기록을 데이터베이스에서 호출하여 현재 질문의 문맥으로 포함함으로써, 앞선 대화의 흐름을 완벽히 이해하는 연속적인 학습 상호작용 구현
